# 專案結構

## 根目錄組織

```
/
├── Backend/     # 伺服器端應用程式
├── Frontend/    # 客戶端應用程式
└── .kiro/       # Kiro 配置和指導規則
```

## 慣例

### Backend/
- 包含所有伺服器端程式碼、API 和業務邏輯
- 應該是自包含且可獨立執行的
- 配置檔案應放在 Backend 根目錄層級
- 將環境特定的配置分開（使用 .env 檔案）

### Frontend/
- 包含所有客戶端程式碼和 UI 元件
- 應該是自包含且可獨立執行的
- 配置檔案應放在 Frontend 根目錄層級
- 保持 API 端點可針對不同環境進行配置

## 一般指南

- 保持 Backend 和 Frontend 完全分離
- 各自應有自己的依賴管理
- 共享的型別/介面可以維護在任一位置或共享資料夾（如有需要）
- 在每個資料夾內使用清晰的命名慣例
- 清楚記錄任何跨資料夾的依賴關係
